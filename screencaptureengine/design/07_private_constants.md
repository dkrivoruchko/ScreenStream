# Screen Capture Engine — Private Constants

## 1. Scope and authority

This document binds the seven private numeric policy constants required by
[05 Gate-B Inputs](05_gate_b_inputs.md#2-private-constants). It is subordinate to the product behavior in
[01 Design](01_design.md), the technical rules in [02 Architecture](02_architecture.md), and the verification
rules in [03 Verification](03_verification.md). A constant selects only when an already-defined boundary is
observed; it does not change the boundary's failure classification, ownership transition, lifecycle priority,
late-cleanup behavior, supported API range, or runtime-path selection.

All deadline values use the injected `EngineClock` time base corresponding to
`SystemClock.elapsedRealtimeNanos()`. They are positive, finite, private implementation policy. They are not a
public timing guarantee or service-level objective. The public callback-entry boundary and Stats publication
cadence remain unchanged and are not restated as private constants here.

It also records the fixed native wire-protocol discriminators required for Kotlin/C interoperability. Those values are not policy knobs, durations, tolerances,
or runtime-selection axes. This document defines no image value or tolerance and grants no implementation authorization.

## 3. Fixed native protocol discriminators

| `NativeResultBlock.writerStatus` meaning | Kotlin `Long` | Native `uint64_t` |
| --- | ---: | ---: |
| `Safe` | `0L` | `0` |
| `OutOfMemory` | `1L` | `1` |
| `InternalFailure` | `2L` | `2` |

Every other 64-bit value is malformed native evidence and maps to `InternalFailure`. These fixed wire values are protocol discriminators, not private safety
policy constants, durations, test tolerances, or additional runtime axes.

## 4. Interpretation and normative links

The five `Long` rows use the injected `EngineClock` and their row-specific checked `S + C` construction; no
deadline may wrap. A complete fact or return is timely only at `T < D`. Equality and every later sample belong to
expiry, while an empty occurrence expires when observed at `now >= D`. The two `Int` rows bound only the exact
error-drain probes and classifications stated in their rows.

Consecutive JPEG stages each arm a fresh occurrence using the existing JPEG value; they create neither a compound
deadline nor a product latency promise or service-level objective.
Target `Surface.release()` and the dependent GL destruction occurrence likewise arm independent Android and GL
intervals in sequence, not a compound deadline or service-level objective.

The authoritative operation/readiness identities, gates, publication points, wake links, watchdog exclusions,
terminal transfer, and cleanup-only late-result mechanics are in
[06 Implementation Design §5](06_implementation_design.md#5-generic-operation-settlement-and-deadlines), with
the concrete owners in [§7](06_implementation_design.md#7-android-capture-metrics-and-target),
[§8](06_implementation_design.md#8-egl-gles2-direct-readback-and-color), and
[§10](06_implementation_design.md#10-jpeg-runtime-framework-path-native-bridge-and-storage). Timeout
visibility and diagnostic noncontrol behavior are owned by
[§12.3](06_implementation_design.md#123-diagnostics).

The executable boundary, race, cap, and diagnostic obligations are mapped in
[08 Test Implementation Map §4](08_test_implementation_map.md#4-test-files-and-owner-coverage) and
[§6](08_test_implementation_map.md#6-deadline-diagnostics-and-noncontrol-behavior). Those documents reference
the exact values and row obligations here rather than redefining them.

These constants authorize no implementation work. Gate B and the separate implementation-authorization
checkpoint remain controlling.
